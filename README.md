# PasswordIOTVerification
## Introduction:
A password verification system contains:
- Password verification.
- A web page.
- Capacity of registering the date of the passwords.

## General Use:
External hardware is needed to use this, an esp32, an lcd display and an 4x4 keypad, the esp32 also needs to be flashed to have the capacity to be programmed with python, Tonny IDE was also used for the development.

First we start the nodejs server that has all the settings for the hash verification, a broker is also needed, in this case mosquitto was used. The service is automatically started, then we run the micropython script with the esp32 connected, is necessary to write the info of the wifi connection that is being used at the moment in the fields ```WIFI_SSID``` and ```WIFI_PASS```. After running the script, the connection with the broker will be made, then we need to run the web page, this web page uses the json generated by the input of the user, it contains all the stored passwords, then it verifies if the hash is correct, if it is, then a message saying allow is shown at the top, on the contrary the message will say that it failed, the page also has the capability to update the password that the user entered on the keypad with choosing and filling the corresponding field, at the end the result will be shown on the lcd display.

## Comunnication Process:
```ESP32``` --> ```MOSQUITTO BROKER``` --> ```NODEJS``` --> ```WEBPAGE VERIFICATION``` --> ```ESP32 CONNECTED DISPLAY```

## Technologies used:
- ESP32 setup with micropython.
- MQTT Broker Mosquitto comunication.
- NodeJs server implementation.
- Hardware usage IOT (LCD Display, 4x4 Keypad)
- JavaScript.
- Html.
- Json Usage.
